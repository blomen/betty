# Run-Gate for Provider Bet Placement — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Selecting a provider on the Play page auto-runs login → balance sync → pending settlement, then idles at a new `STATE_READY_TO_RUN` until the user clicks the card again to toggle into the bet-placement loop.

**Architecture:** Add a per-runner `_run_event` (`asyncio.Event`) gate between the existing settle step and the bet loop in both `ProviderRunner` and `ArbRunner`. The card click handler in `PlayPage.tsx` becomes state-aware: on the new "ready" state it toggles the gate via two new endpoints (`POST /mirror/play/run/{pid}` and `POST /mirror/play/pause/{pid}`). A continuous-sync task spawned during ready re-runs balance fetch + pending detection on a slow cadence and is cancelled when the runner enters its bet loop.

**Tech Stack:** Python 3.10+ asyncio (FastAPI route + asyncio.Event gate), React 19 + TypeScript (color-mapped card states), Vite (rebuild via `arnold.bat`).

**Test note:** This codebase has no existing pytest coverage for `play_loop.py` / `provider_runner.py` / `arb_runner.py` (those files are untested today). We add unit tests for the new gate-state machine in pure-Python form, no live browser. No frontend test framework is configured for `arnold/frontend/` — frontend changes are verified manually per Task 8's checklist.

    def set_run(self, run: bool) -> bool:
        """Toggle the run gate. Returns True if state actually changed.

        Setting True while at STATE_READY_TO_RUN releases the gate so the
        bet loop iteration can begin. Clearing while running causes the
        loop to drop back to STATE_READY_TO_RUN at the top of the next
        iteration (mid-bet placements unwind naturally — pausing while at
        STATE_READY waiting for Place/Skip simulates a Skip)."""

                return False
            self._run_event.set()
            return True
        else:
            if not self._run_event.is_set():
                return False

            # If runner is sitting at STATE_READY waiting on Place/Skip,
            # signal Skip so the bet loop can unwind and re-check the gate.
            if self.state == STATE_READY:
                self._skip_event.set()
            return True

```python
    async def _ready_sync_loop(self, workflow: Any, page: Any) -> None:
        """Continuously refresh balance and re-detect pending while at
        STATE_READY_TO_RUN. Cancelled when the runner enters its bet loop
        or stops. Errors are swallowed — staleness is preferable to crashing
        the runner mid-idle."""
        pid = self.provider_id
        last_balance = 0.0
        last_pending = 0.0
        while True:
            now = asyncio.get_event_loop().time()
            try:
                if now - last_balance >= READY_BALANCE_SYNC_INTERVAL_S:
                    if hasattr(workflow, "fetch_balance"):
                        try:
                            await workflow.fetch_balance(page)
                        except Exception as e:
                            logger.debug(f"[Runner:{pid}] ready balance sync failed: {e!r}")
                    last_balance = now
                if now - last_pending >= READY_PENDING_SYNC_INTERVAL_S:
                    try:
                        await self._detect_pending(pid, workflow, page)
                        # _detect_pending sets self.state = STATE_SETTLING; restore
                        self.state = STATE_READY_TO_RUN
                    except Exception as e:
                        logger.debug(f"[Runner:{pid}] ready pending sync failed: {e!r}")
                    last_pending = now
            except asyncio.CancelledError:
                raise
            except Exception:
                pass
            await asyncio.sleep(5.0)
```

    @router.post("/play/run/{provider_id}")
    async def play_run(provider_id: str):
        """Release the Run gate for a provider runner: yellow → green.
        409 if no runner exists for this provider."""
        ok = play_loop.set_run(provider_id, True)
        if not ok:
            raise HTTPException(
                status_code=409,
                detail=f"No runner for {provider_id} or gate already open",
            )
        return play_loop.get_status()

    @router.post("/play/pause/{provider_id}")
    async def play_pause(provider_id: str):
        """Clear the Run gate for a provider runner: green → yellow.
        409 if no runner exists or gate already closed."""
        ok = play_loop.set_run(provider_id, False)
        if not ok:
            raise HTTPException(
                status_code=409,
                detail=f"No runner for {provider_id} or gate already closed",
            )
        return play_loop.get_status()
```

that re-checks it — and verify the contract:

  1. Gate starts cleared.
  2. set_run(True) flips False→True idempotently (returns False on no-op).
  3. set_run(False) flips True→False idempotently.
  4. A waiter awaiting the event unblocks when set.
  5. A loop that re-checks the event drops back to ready when cleared.

Plus state-constant assertions to lock in the public surface.
"""
from __future__ import annotations

import asyncio

import pytest

from arnold.mirror.play_loop import STATE_READY_TO_RUN


def test_state_constant_value():
    """STATE_READY_TO_RUN is a stable string the frontend depends on."""
    assert STATE_READY_TO_RUN == "ready_to_run"


class _GateHarness:
    """Minimal mirror of the gate primitive used by both runners."""

    def __init__(self):
        self._run_event = asyncio.Event()

    def set_run(self, run: bool) -> bool:
        if run:
            if self._run_event.is_set():
                return False
            self._run_event.set()
            return True
        else:
            if not self._run_event.is_set():
                return False
            self._run_event.clear()
            return True


def test_gate_starts_cleared():
    h = _GateHarness()
    assert not h._run_event.is_set()


def test_set_run_true_flips_and_is_idempotent():
    h = _GateHarness()
    assert h.set_run(True) is True
    assert h._run_event.is_set()
    assert h.set_run(True) is False  # already set


def test_set_run_false_clears_and_is_idempotent():
    h = _GateHarness()
    h.set_run(True)
    assert h.set_run(False) is True
    assert not h._run_event.is_set()
    assert h.set_run(False) is False  # already cleared


@pytest.mark.asyncio
async def test_waiter_unblocks_on_set():
    h = _GateHarness()

    async def waiter():
        await h._run_event.wait()
        return "released"

    task = asyncio.create_task(waiter())
    await asyncio.sleep(0.01)  # let the waiter park
    assert not task.done()
    h.set_run(True)
    result = await asyncio.wait_for(task, timeout=1.0)
    assert result == "released"


@pytest.mark.asyncio
async def test_loop_drops_back_to_ready_when_cleared():
    """A bet-loop that re-checks the gate at iteration start should park
    on the event when cleared, and resume on set."""
    h = _GateHarness()
    h.set_run(True)
    iterations: list[str] = []

    async def bet_loop():
        for _ in range(3):
            if not h._run_event.is_set():
                iterations.append("waiting")
                await h._run_event.wait()
            iterations.append("ran")

    task = asyncio.create_task(bet_loop())
    await asyncio.sleep(0.01)  # one iteration completes
    h.set_run(False)
    await asyncio.sleep(0.01)  # next iteration parks
    h.set_run(True)
    await asyncio.wait_for(task, timeout=1.0)
    # First iter ran, second iter waited then ran, third iter ran.
    assert iterations == ["ran", "waiting", "ran", "ran"]


def test_play_loop_set_run_no_runner_returns_false():
    """PlayLoop.set_run(pid) on an unknown provider returns False, never raises."""
    from arnold.mirror.browser import MirrorBrowser
    from arnold.mirror.play_loop import PlayLoop
    from arnold.mirror.sse import MirrorBroadcaster

    pl = PlayLoop(
        browser=MirrorBrowser.__new__(MirrorBrowser),  # uninitialized — never used here
        broadcaster=MirrorBroadcaster.__new__(MirrorBroadcaster),
        proxy_url="http://localhost:8000",
    )
    # No runners registered — set_run is a no-op that returns False.
    assert pl.set_run("nonexistent_provider", True) is False
    assert pl.set_run("nonexistent_provider", False) is False
```

## Notes on Continuous Sync

`workflow.fetch_balance` does not exist on every workflow base class; some workflows expose balance only via the network interceptor. If `hasattr(workflow, "fetch_balance")` is False (which is the runtime guard already in the code), balance simply doesn't refresh while idle on those providers — the interceptor still updates balance live whenever the user interacts with the provider site. This is a graceful degradation, not a bug. Pending detection (`_detect_pending`) is implemented on the runner itself, so it works for every provider.
