→

postmortem table
Recompute scope: On settlement → single bet/trade. On manual recompute → all settled bets/trades missing postmortem or where version < current algo version.
Scheduler: No automatic schedule needed. Runs inline after settlement (async, non-blocking). Manual recompute for bulk reclassification after algo changes.

API Endpoints

New routes in backend/src/api/routes/
GET  /api/postmortem/bets — classified bets with filters (classification, market, provider, date range)
GET  /api/postmortem/bets/summary — aggregate stats by classification
GET  /api/postmortem/bets/patterns — auto-detected pattern insights
GET  /api/postmortem/trades — classified trades with filters
GET  /api/postmortem/trades/summary — aggregate stats by classification
GET  /api/postmortem/trades/patterns — auto-detected pattern insights
POST /api/postmortem/recompute — force recompute all (after algo update)
