Pattern Detection Engine

How insights are automatically surfaced from postmortem data

Segmentation Dimensions

The engine slices your results across these axes and looks for statistically significant deviations

Betting Segments
By market: 1x2 / spread / total
By provider: each soft book individually
By sport: football, basketball, hockey, etc.
By edge band: <2% / 2-5% / 5-10% / 10%+
By TTK band: <6h / 6-24h / 24-48h / 48h+
By odds range: <1.5 / 1.5-2.5 / 2.5-4.0 / 4.0+
By day of week: Mon-Sun
By classification: the 6 postmortem categories
Trading Segments
By setup type: VWAP bounce, breakout, etc.
By instrument: NQ, ES, etc.
By direction: long / short
By session: pre-market / RTH / post
By streak: after N consecutive wins/losses
By psych score: <5 / 5-7 / 7+
By day of week: Mon-Fri
By R-multiple band: <-1R / -1-0R / 0-1R / 1-2R / 2R+

Pattern Detection Rules

Each rule checks a segment for deviation from baseline, only surfaces if sample ≥ 10

PatternDetector — rules engine
Rule Trigger Icon
Losing segment ROI < -10% in segment with n ≥ 10 ▼
Winning segment ROI > +5% in segment with n ≥ 10 ▲
Edge erosion hotspot ≥40% of losses classified as edge_erosion in a TTK/provider segment ●
Sizing alert ≥3 sizing_error classifications in trailing 30 days ●
Streak impact (trading) Win rate after N losses deviates >15pp from baseline ▼
Psych correlation (trading) Avg R differs >0.5R between psych score bands ●
Setup underperformer (trading) Setup type avg R < 0 with n ≥ 5 ▼
False edge concentration ≥30% of losses in a provider+market segment are false_edge ▼
